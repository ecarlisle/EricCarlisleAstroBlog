Container queries are the most important CSS feature since Grid. They let components respond to their own size rather than the viewport, which is what we actually wanted all along.

## The Mental Shift

With media queries, you think in terms of page layouts. With container queries, you think in terms of component layouts. A card component styles itself differently depending on how much room it has, regardless of whether it's in a sidebar or a full-width section.

```css
.card-wrapper {
  container-type: inline-size;
  container-name: card;
}

@container card (min-width: 400px) {
  .card {
    display: grid;
    grid-template-columns: 200px 1fr;
  }
}
```

## Container Query Units

cqw, cqh, cqi, cqmin, cqmax — these are relative to the container rather than the viewport. They're incredibly useful for typography that scales with its container:

```css
.card-title {
  font-size: clamp(1rem, 4cqi, 2rem);
}
```

## Browser Support and Fallbacks

Container queries are well-supported now, but if you need to support older browsers, use a `@supports` query:

```css
.card {
  /* Fallback: stacked layout */
  display: block;
}

@supports (container-type: inline-size) {
  .card-wrapper {
    container-type: inline-size;
  }

  @container (min-width: 400px) {
    .card {
      display: grid;
      grid-template-columns: 200px 1fr;
    }
  }
}
```

The fallback is just a simpler layout. No harm done.
