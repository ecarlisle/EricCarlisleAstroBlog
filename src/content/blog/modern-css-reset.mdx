The classic reset — `* { margin: 0; box-sizing: border-box; }` — is a start, but it leaves too many inconsistencies on the table. Browsers have converged on defaults in the last few years, which means our resets can be more targeted.

## The Core

```css
*,
*::before,
*::after {
  box-sizing: border-box;
}

body {
  margin: 0;
  min-height: 100vh;
  line-height: 1.5;
  -webkit-font-smoothing: antialiased;
}

img, picture, video, canvas, svg {
  display: block;
  max-width: 100%;
}

input, button, textarea, select {
  font: inherit;
}
```

## Why These Matter

`line-height: 1.5` on the body sets a reasonable rhythm everywhere. `font: inherit` on form elements prevents them from defaulting to the browser's system font. Block display on images eliminates the mysterious 4px gap below them.

## Reduced Motion

Add a reduced-motion layer at the reset level so you don't have to remember it in every component:

```css
@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
```

This isn't elegant, but it's effective. The `!important` is justified here — it's a user preference override.
