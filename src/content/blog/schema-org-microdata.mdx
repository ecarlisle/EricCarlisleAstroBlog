Every page on this site emits a `<script type="application/ld+json">` block rendered by `SchemaOrg.astro`. The contents of that block vary by page type — a homepage gets different entities than an article or an about page.

This article documents every schema graph that ships to search engines, so you know exactly what structured data is available for indexing, rich results, and knowledge graph connections.

## The Author Entity (shared across all pages)

All page types include the author `Person` entity as the first item in the `@graph` array. It links back to the same `@id` everywhere so search engines resolve it as one identity:

```json
{
  "@type": "Person",
  "@id": "https://ericcarlisle.com/#person",
  "name": "Eric Carlisle",
  "url": "https://ericcarlisle.com",
  "image": {
    "@type": "ImageObject",
    "@id": "https://ericcarlisle.com/#personImage",
    "url": "https://ericcarlisle.com/eric-carlisle.webp"
  },
  "sameAs": [
    "https://linkedin.com/in/ericcarlisle",
    "https://github.com/ecarlisle"
  ],
  "jobTitle": "Principal Full-Stack Engineer & UX Architect",
  "worksFor": {
    "@type": "Organization",
    "name": "Eric Carlisle Consulting"
  }
}
```

## Homepage (`pageType="homepage"`)

The index page at `/` produces three entities in its `@graph`:

1. **`Person`** — author identity (see above).
2. **`WebSite`** — site-level metadata with a `SearchAction` potential action for Google's Sitelinks Search Box:
   ```json
   {
     "@type": "WebSite",
     "@id": "https://ericcarlisle.com/#website",
     "url": "https://ericcarlisle.com",
     "name": "Eric Carlisle",
     "publisher": { "@id": "https://ericcarlisle.com/#person" },
     "potentialAction": [{
       "@type": "SearchAction",
       "target": {
         "@type": "EntryPoint",
         "urlTemplate": "https://ericcarlisle.com/search?q={search_term_string}"
       },
       "query-input": "required name=search_term_string"
     }]
   }
   ```
3. **`CollectionPage`** — the page itself, linked to the website via `isPartOf`.

The `WebSite` entity only appears on the homepage — no other page type includes it. The `SearchAction` is how Google discovers your site search and can render a search box in search results.

## Blog Index & Paginated Pages (`pageType="collection"`)

Every collection page — including `/blog`, `/blog/page/2`, and the 404 page — emits a `CollectionPage` entity with a `BreadcrumbList`:

```json
{
  "@type": "BreadcrumbList",
  "@id": "https://ericcarlisle.com/blog/#breadcrumb",
  "itemListElement": [
    { "@type": "ListItem", "position": 1, "name": "Eric Carlisle", "item": "https://ericcarlisle.com" },
    { "@type": "ListItem", "position": 2, "name": "Blog", "item": "https://ericcarlisle.com/blog" }
  ]
}
```

When posts are passed to `BaseHead`, the `CollectionPage` also includes a `mainEntity`:

```json
"mainEntity": {
  "@type": "ItemList",
  "numberOfItems": 10,
  "itemListElement": [
    { "@type": "ListItem", "position": 1, "name": "...", "url": "..." }
  ]
}
```

The `collectionType` prop distinguishes three variants:

| collectionType | Breadcrumb | Notes |
|----------------|------------|-------|
| `main-feed` | Home → Blog | Standard blog index |
| `tag-filter` | Home → Tags → TagName | Filtered by tag |
| `tags-index` | Home → Tags | Tag archive overview |

The current pages that use `pageType="collection"` are:

| Route | collectionType | Has posts list? |
|-------|---------------|-----------------|
| `/blog` | `main-feed` (default) | Yes |
| `/blog/page/2` | `main-feed` (default) | Yes |
| `/404` | `main-feed` (default) | No (`robots: "noindex, follow"`) |

## Article Pages (`pageType="article"`)

Every blog post at `/blog/[...slug]/` produces a graph with:

1. **`Person`** — author.
2. **`WebPage`** — the page itself, with `mainEntity` pointing to the article.
3. **`BlogPosting`** — the full article schema.
4. **`BreadcrumbList`** — Home → Blog → Post Title.

The `BreadcrumbList` for articles has three items:

```json
{
  "@type": "BreadcrumbList",
  "@id": "https://ericcarlisle.com/blog/some-post/#breadcrumb",
  "itemListElement": [
    { "@type": "ListItem", "position": 1, "name": "Eric Carlisle", "item": "https://ericcarlisle.com" },
    { "@type": "ListItem", "position": 2, "name": "Blog", "item": "https://ericcarlisle.com/blog" },
    { "@type": "ListItem", "position": 3, "name": "Post Title", "item": "https://ericcarlisle.com/blog/some-post/" }
  ]
}
```

The `BlogPosting` entity drives search-result appearance — rich snippets, Google Discover, and knowledge panels. Key properties:

```json
{
  "@type": "BlogPosting",
  "@id": "https://ericcarlisle.com/blog/some-post/#article",
  "mainEntityOfPage": { "@id": "https://ericcarlisle.com/blog/some-post/#webpage" },
  "headline": "Post Title",
  "description": "Post description",
  "datePublished": "2026-06-17T00:00:00.000Z",
  "dateModified": "2026-06-17T00:00:00.000Z",
  "articleSection": "first-tag",
  "wordCount": 850,
  "timeRequired": "PT5M",
  "author": { "@type": "Person", "name": "Eric Carlisle", "@id": "https://ericcarlisle.com/#person" },
  "publisher": { "@type": "Person", "name": "Eric Carlisle", "@id": "https://ericcarlisle.com/#person" },
  "image": { "@type": "ImageObject", "@id": ".../#primaryimage", "url": "...", "width": 1200, "height": 630 },
  "inLanguage": "en-US"
}
```

Properties are conditionally included:

| Property | Condition |
|----------|-----------|
| `dateModified` | Only when `updatedDate` is set in frontmatter |
| `articleSection` | Only when `tags` has at least one entry |
| `wordCount`, `timeRequired` | Only when `wordCount > 0` |
| `image` | Always — falls back to `og-image.jpg` |

Reading time is computed as `Math.ceil(wordCount / 200)` and represented as ISO 8601 duration (`PT5M`).

Open Graph and Twitter meta tags also vary by article metadata:

| Tag | Source |
|-----|--------|
| `article:published_time` | `pubDate` |
| `article:modified_time` | `updatedDate` (optional) |
| `article:author` | "Eric Carlisle" |
| `article:section` | First tag |
| `article:tag` | Each tag (repeated per tag) |
| `og:type` | `article` |
| `twitter:creator` | `twitterHandle` from frontmatter (optional) |

## About & Contact Pages (`pageType="about"` / `pageType="contact"`)

These produce a graph with:

1. **`Person`** — author.
2. **`AboutPage`** or **`ContactPage`** — the page itself, with `mainEntity` pointing to `/#person`.
3. **`BreadcrumbList`** — Home → Page Title.

The `AboutPage` entity:

```json
{
  "@type": "AboutPage",
  "@id": "https://ericcarlisle.com/about/#webpage",
  "url": "https://ericcarlisle.com/about/",
  "name": "About Me",
  "description": "...",
  "mainEntity": { "@id": "https://ericcarlisle.com/#person" }
}
```

Pages that use these types:

| Route | pageType |
|-------|----------|
| `/about` | `about` |
| `/style-guide` | `about` (via BlogPost layout) |

## Page Type Summary

| pageType | Schema Types | Breadcrumb |
|----------|-------------|------------|
| `homepage` | `Person`, `WebSite`, `CollectionPage` | — |
| `collection` | `Person`, `CollectionPage`, `ItemList` | Home → Blog |
| `article` | `Person`, `WebPage`, `BlogPosting`, `BreadcrumbList` | Home → Blog → Post |
| `about` | `Person`, `AboutPage`, `BreadcrumbList` | Home → About |
| `contact` | `Person`, `ContactPage`, `BreadcrumbList` | Home → Contact |
| `portfolio` | `Person`, `WebPage`, `CreativeWork`, `BreadcrumbList` | Home → Portfolio |

All types also render Open Graph, Twitter Card, and standard meta tags via `BaseHead.astro`. The full `@graph` is rendered inside a single `<script type="application/ld+json">` element, placed just before `</head>`.

## Debugging Structured Data

To inspect the live schema on any page:

1. Open **Chrome DevTools → Elements** and search for `ld+json`.
2. Use **Google's [Rich Results Test](https://search.google.com/test/rich-results)** to validate the markup.
3. Use **Schema.org's [Validator](https://validator.schema.org/)** to trace `@id` references and verify entity relationships.

Every article page should pass the Rich Results Test with zero errors. If it doesn't, the most common cause is a missing or invalid `heroImage` path, since `image` is always included in the `BlogPosting` entity.
