Forms are the most common interaction on the web, and they're also where accessibility failures cluster. The good news is that fixing them doesn't require heroics — just attention to a few patterns.

## Label Everything

Every input needs a `<label>`. Placeholders are not labels — they disappear on input, leaving users with no context.

```html
<label for="email">Email address</label>
<input type="email" id="email" name="email" autocomplete="email" />
```

## Error Messages That Work

Inline validation is best, but it needs to be connected to its input programmatically:

```html
<label for="name">Full name</label>
<input
  type="text"
  id="name"
  name="name"
  aria-describedby="name-error"
  aria-invalid="true"
/>
<span id="name-error" role="alert">Name is required.</span>
```

`aria-describedby` ties the error to the input so screen readers announce it when focusing. `role="alert"` makes the error message interrupt politely.

## Focus Management After Submission

After a form submission with errors, move focus to the first invalid input. This prevents keyboard users from having to tab through the entire form again to find what went wrong.

```js
const firstError = form.querySelector('[aria-invalid="true"]');
if (firstError) firstError.focus();
```

## Testing

Run your form through axe DevTools, then navigate it with a keyboard. If you can complete the flow with only Tab, Enter, and Escape, you're 90% of the way there.
