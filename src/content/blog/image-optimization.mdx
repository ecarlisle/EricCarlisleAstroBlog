Images are the largest resource on most web pages, and they're the easiest place to save hundreds of kilobytes with almost no visible quality loss.

## Modern Formats

AVIF provides the best compression today — roughly 50% smaller than JPEG at the same perceptual quality. WebP is a safe fallback with broader support. Serve both and let the browser choose:

```html
<picture>
  <source type="image/avif" srcset="/hero.avif" />
  <source type="image/webp" srcset="/hero.webp" />
  <img src="/hero.jpg" alt="Hero image" width="1200" height="600" />
</picture>
```

## Responsive Sizes

Serving a 2400px image to a 375px viewport wastes bandwidth. Use `srcset` with multiple widths and `sizes` to tell the browser which one to pick:

```html
<img
  src="/hero-1200.jpg"
  srcset="/hero-400.jpg 400w, /hero-800.jpg 800w, /hero-1200.jpg 1200w"
  sizes="(max-width: 720px) 100vw, 1200px"
  width="1200"
  height="600"
  loading="lazy"
/>
```

## Automation

Do not hand-edit images. Use a build pipeline — Astro's built-in image optimization, Sharp, or a service like Cloudinary — that generates all formats and sizes from a single source. The build step adds a few seconds; the bandwidth savings last forever.

Astro's `<Image />` and `<Picture />` components handle this automatically. Use them instead of raw `<img>` tags wherever possible.

## The One Rule

Every image should have explicit `width` and `height` attributes. This prevents Cumulative Layout Shift and lets the browser reserve space before the image loads. It's the single highest-impact thing you can do for image performance.
