Every visual decision on this site flows through CSS custom properties defined in a single file: `src/styles/global.css`. This article documents those tokens, their relationships, and the rationale behind them.

## Typography

### Font Families

```css
--font-copy: "Inter Variable", system-ui, sans-serif;
--font-headers: "Plus Jakarta Sans Variable", system-ui, sans-serif;
--font-mono: "Fira Code Variable", ui-monospace, monospace;
```

Inter is the workhorse body face — narrow, tall x-height, excellent at small sizes. Plus Jakarta Sans handles headings with a more geometric, open character. Fira Code provides monospace with ligatures.

### Type Size Scale

Every size is fluid — it scales smoothly between viewport widths using `clamp()`:

```css
--type-size-body:   clamp(1rem,    0.45vw + 0.91rem, 1.25rem);   /* 16px → 20px */
--type-size-small:  clamp(0.8125rem, 0.2vw + 0.77rem, 0.9375rem); /* 13px → 15px */
--type-size-h1:     clamp(2rem,    3.64vw + 1.27rem,  4rem);      /* 32px → 64px */
--type-size-h2:     clamp(1.625rem, 2.13vw + 1.2rem, 2.8rem);    /* 26px → 45px */
--type-size-h3:     clamp(1.375rem, 1.14vw + 1.15rem, 2rem);     /* 22px → 32px */
--type-size-h4:     clamp(1.125rem, 0.57vw + 1.07rem, 1.5rem);   /* 18px → 24px */
--type-size-h5:     clamp(1rem,    0.28vw + 0.94rem, 1.1875rem); /* 16px → 19px */
--type-size-blockquote: 1.1em;
```

The scale uses a roughly 1.25 modular ratio (major third). The `clamp()` endpoints are chosen so that body text never drops below 16 px (the minimum for comfortable long-form reading on a 13″ laptop) and never exceeds it by more than 25 %.

### Line Heights

```css
--line-height-body: 1.65;    /* body copy — generous for readability */
```

There is no separate heading line-height token. Each heading level sets its own line-height as a multiple of the vertical rhythm unit (see below).

## Vertical Rhythm

The entire vertical spacing system is anchored to one value:

```css
--rhythm: calc(var(--line-height-body) * 1rem);  /* ≈ 1.65 rem */
```

`--rhythm` equals one line of body text. Every margin, padding, and heading line-height is a clean multiple of this unit, creating a consistent baseline grid.

### Heading Sizes and Spacing

| Heading | Font size (fluid) | Line-height (× rhythm) | Top margin | Bottom margin |
|---------|-------------------|------------------------|------------|---------------|
| h1      | 32 → 64 px        | 3× (≈ 79 px)           | 0          | 1× (≈ 26 px)  |
| h2      | 26 → 45 px        | 2× (≈ 53 px)           | 2×         | 1×            |
| h3      | 22 → 32 px        | 2× (≈ 53 px)           | 1.5×       | 0.5×          |
| h4      | 18 → 24 px        | 1.5× (≈ 40 px)         | 1.5×       | 0.5×          |
| h5      | 16 → 19 px        | 1.25× (≈ 33 px)        | 1×         | 0.5×          |

The bottom margin of a heading is always smaller than its top margin, so each heading visually attaches to the content below it rather than floating between sections.

## Spacing Scale

The spacing tokens provide the building blocks for layout, padding, and gaps. Every value is a multiple of `--rhythm`:

```css
--space-xs: calc(var(--rhythm) * 0.5);   /*  0.5× ≈ 13 px */
--space-sm: calc(var(--rhythm) * 0.75);  /*  0.75× ≈ 20 px */
--space-md: calc(var(--rhythm) * 1);     /*  1×   ≈ 26 px */
--space-lg: calc(var(--rhythm) * 1.5);   /*  1.5× ≈ 40 px */
--space-xl: calc(var(--rhythm) * 2);     /*  2×   ≈ 53 px */
```

These are further aliased by purpose so that changing one token adjusts spacing everywhere it is used:

```css
--space-inline: var(--space-xs);            /* tight inline gaps (tag spacing, icon gaps) */
--space-inline-strong: var(--space-sm);      /* stronger inline separation (card children) */
--space-component: var(--space-md);          /* default component padding */
--space-component-lg: var(--space-lg);        /* large component padding */
--space-section: var(--space-lg);            /* section-level margins (headings, blockquotes) */
```

## Color Palette

The colors use the OKLCH color space for perceptual uniformity. The default theme is a warm dark mode with an amber/coral accent palette.

### Dark Theme (default)

```css
--bg-main: oklch(0.12 0.015 280);             /* deep navy-black background */
--bg-surface: oklch(0.16 0.02 280);           /* card / surface background */
--bg-surface-elevated: oklch(0.2 0.025 280);  /* elevated surface (hovered cards) */

--text-primary: oklch(0.95 0.008 280);        /* main body text */
--text-secondary: oklch(0.75 0.015 280);      /* secondary text */
--text-muted: oklch(0.58 0.02 280);           /* muted / meta text */

--brand-primary: oklch(0.7 0.18 70);          /* amber brand */
--brand-accent: oklch(0.65 0.2 30);           /* coral accent links */
--brand-highlight: oklch(0.85 0.14 90);       /* highlight / hover */
```

### Light Theme Override

The light theme is opt-in via `html[data-theme="light"]` and overrides only the color tokens. The typography and spacing tokens stay the same:

```css
[data-theme="light"] {
  --bg-main: oklch(0.985 0.003 280);           /* near-white background */
  --bg-surface: oklch(1 0 0);                  /* pure white cards */
  --bg-surface-elevated: oklch(0.92 0.008 280);
  /* ... text and brand tokens shift to darker values */
}
```

Using OKLCH ensures that the light and dark themes stay perceptually consistent — a `--text-muted` of 0.58 lightness reads the same way in both themes even though the absolute luminance differs.

## Border Radii

```css
--radius-sm: 4px;
--radius-md: 8px;
--radius-lg: 12px;
```

A tight radius system. Cards use `--radius-lg`, buttons and inputs use `--radius-md`, code blocks use `--radius-sm`.

## Motion

```css
--transition-fast: 0.15s ease-out;
--transition-normal: 0.25s ease-out;
--transition-slow: 0.4s ease-out;
```

All transitions use `ease-out` — fast in, slow out — which feels responsive because the motion starts immediately and coasts to rest. The `prefers-reduced-motion: reduce` media query disables all transitions and animations.

## Structural Tokens

```css
--size-border-pixel: 2px;
--size-border-thick: 4px;
--size-touch-target-min: 44px;

--layout-max-width: 68ch;           /* comfortable reading measure */
--layout-max-width-wide: 75rem;     /* full page content area */
--bp-mobile: 720px;                 /* mobile breakpoint */
```

The 68-character line length for body text follows the established range of 45–75 characters per line. The touch target minimum of 44px matches [WCAG 2.1 Success Criterion 2.5.8](https://www.w3.org/TR/WCAG21/#target-size).
