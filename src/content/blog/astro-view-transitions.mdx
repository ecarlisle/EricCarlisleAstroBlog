Astro's View Transitions API brings native-like page transitions to multi-page applications without a heavy client-side framework. It's built on the browser's View Transitions API with Astro-specific enhancements.

## Getting Started

Add the component to your layout head:

```astro
---
import { ViewTransitions } from 'astro:transitions';
---
<html>
  <head>
    <ViewTransitions />
  </head>
</html>
```

That's it. Every navigation between pages now transitions with a cross-fade by default.

## Named Elements for Morphing

The real power comes from `transition:name` and `transition:animate`. Elements with the same name across pages morph smoothly:

```astro
<!-- On the blog index -->
<img src={post.image} alt={post.title} transition:name={post.slug} />

<!-- On the individual post page -->
<img src={post.image} alt={post.title} transition:name={post.slug} transition:animate="morph" />
```

The image scales and morphs between its position on the index and its position on the detail page. No JS required.

## Preserving Client State

Frameworks like React or Alpine maintain their state across navigations if you use `transition:persist`:

```astro
<Header transition:persist />
```

The header DOM element is reused across navigations, so open menus, scroll position, and hover states survive.

## Caveats

View transitions require the navigation to be same-origin. External links and form submissions aren't intercepted. Also, the animations run on the compositor thread, so they won't block interaction — but they can look janky if the pages have very different layouts. Use `transition:animate="morph"` selectively.
