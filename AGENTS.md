> Claude Code users: see [CLAUDE.md](./CLAUDE.md) for full guidance.

## Project

Astro 6 static blog — Markdown/MDX content collections, RSS, sitemap, local fonts.

- Package manager: **pnpm** (required — do not use npm/yarn)
- Node: `>=22.12.0`

## Commands

```sh
pnpm dev            # dev server
pnpm build          # production build
pnpm preview        # preview build output
pnpm astro check    # type-check .astro files
pnpm exec vitest    # run tests
pnpm exec biome check .           # lint
pnpm exec biome format . --write  # format
```

No `lint`/`test`/`format` package scripts — always use `pnpm exec` directly.

## Key Files

| Path | Purpose |
|------|---------|
| `astro.config.mjs` | Integrations (MDX, sitemap), local font config |
| `src/content.config.ts` | **Authoritative** blog frontmatter schema (Zod) |
| `src/styles/global.css` | Design tokens — use these, not hardcoded values |
| `src/styles/components.css` | Shared component helpers (`.card`, `.tag`, `.callout`, …) |
| `src/layouts/BlogPost.astro` | Wraps every post; hero image, metadata, header/footer |
| `src/components/BaseHead.astro` | Canonical URL, OG/Twitter tags, font loading |
| `docs/design_system_specification.md` | Token naming reference |

## Architecture

- **Routing**: file-based under `src/pages/`; `[...slug].astro` renders posts
- **Content**: `getCollection('blog')` from `src/content/blog/**/*.{md,mdx}`
- **RSS/Sitemap**: auto-generated by integrations; no manual maintenance needed

## Design Rules

- Dark mode is the **default**; light mode activates via `html[data-theme="light"]`
- Use CSS custom properties from `global.css` — no hardcoded colors, px/rem when tokens exist
- Fluid spacing tokens: `--space-xs` / `--space-sm` / `--space-md` / `--space-lg` / `--space-xl`
- Respect `@media (prefers-reduced-motion: reduce)` for animations
- 44px minimum touch targets; preserve `:focus-visible` on interactive elements

## Non-Obvious Pitfalls

1. **Docs ≠ live tokens**: `design_system_specification.md` describes `--step-1` / `--step-2` naming; `global.css` uses `--space-xs`/`--space-sm` naming. Match `global.css` when editing styles.
2. **Site URL mismatch**: `astro.config.mjs` sets `site: 'https://example.com'`; `vitest.config.ts` overrides to `https://ericcarlisle.com`. Don't rely on `Astro.site` in tests.
3. **Frontmatter schema**: Use `src/content.config.ts` as the single source of truth — the docs contain an outdated snippet.
4. **No theme toggle component**: `data-theme="light"` switching exists in CSS but no UI component implements it yet.
